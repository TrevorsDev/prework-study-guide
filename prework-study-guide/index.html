Git

    What is Git?

  • Git is a version control system that stores your code and its history, as well as separate versions of the code.
  • Git is a system that allows you to create "save points" (called commits) of your work. It's good practice to commit your work whenever you reach a good stopping point. This creates a history of changes and allows you to revert to an earlier version if necessary.

    • Git Flow

    • Follow the git flow steps to add, commit, and push our work to our feature branch.
    • Then we open a pull request with my feature branch as the comparison branch and the default branch as the base branch, merge changes with the main branch, and close the associated GitHub issue.
  • "" .references the current directory, "" .. references the directory above the current one, and cd "" will move you forward in a directory
  • Git status: checks what branch we are currently on
  • The git command must precede every Git statement.
  • The checkout command is to move the working branch to a new branch.
  • Git checkout -b branch-name: creates a new branch and switches to it.
  • The "branch name", ex: 'feature/starter-code' is the name of the new branch. We have added the prefix, feature, to indicate that it is a feature. It is good practice to name the branches for the feature that will be developed by them to help indicate the purpose of each branch.
  • To code to the current working branch, ex: 'feature/starter-code', we'll need to do the following:
    • git add -A
      • The add command adds modifications in the current working branch to the staging area.
      • The -A flag indicates that we want to add all changes.
    • Now that we have staged those files to be committed, let’s enter the git commit command with a commit message that describes what the commit contains. Ex:
      • git commit -m "add starter code"
    • Before we push this up to our repo, we should double-check if our local branch is in sync with the base branch in GitHub. So let’s pull down any changes from the base branch.:
      • git pull origin main
  • In Git, a commit is a snapshot of the changes made to the codebase at a specific point in time. A commit is a permanent and unchangeable record of the changes made to the codebase. A commit represents a single point in the history of the codebase.A stage, on the other hand, is a temporary holding area for changes that have been made to the codebase, but have not yet been committed. When you stage changes in Git, you are preparing them for a commit, but you have not yet made the commit. This allows you to review your changes and make sure they are correct before committing them.You can think of a commit as a final version of the changes and a stage as a draft version of the changes.

JavaScript

  • In web development, JavaScript is the tool that takes a static HTML/CSS webpage and makes it interactive. Every time you do something with a webpage, such as click a button, retrieve data, or fill out a form, JavaScript is at work!
  • The Console
  • The console is a panel that displays important messages, like errors, for developers.
  • The console.log method was created to make it easier for developers to test their code. Developers use it daily to debug and test the code that they write.
  • The console keyword refers to an object, a collection of data and actions, that we can use in our code.
  • Keywords are words that are built into the JavaScript language, so the computer recognizes them and treats them specially.
  • One action, or method, that is built into the console object is the .log() method, which means a set of instructions that can be executed by a computer. .log() will output whats inside the parentheses to the console.
  • Comments
  • Single line comments are great for adding context to your code. Multi-line comments are often best suited to prevent a block of code from running.

    Properties

  • When you introduce a new piece of data into a JavaScript program, the browser saves it as an instance of the data type.
  • All data types have access to specific properties that are passed down to each instance.
  • For example, every string instance has a property called length that stores the number of characters in that string.
  • You can retrieve property information by appending the string with a period and the property name:
    • console.log('Hello'.length); // Prints 5
    • In the example above, the value saved to the length property is retrieved from the instance of the string, 'Hello'

    Logical Operators

  • Working with conditionals means that we will be using booleans, true or false values.
  • In JavaScript, there are operators that work with boolean values known as logical operators.
  • We can use logical operators to add more sophisticated logic to our conditionals.
    • the and operator (&&) - checks that two things are true.
    • the or operator (||) - If we only care about either condition being true
    • the not operator, otherwise known as the bang operator (!) - reverses, or negates, the value of a boolean:
      • Essentially, the ! operator will either take a true value and pass back false, or it will take a false value and pass back true.

    • Conditionals

  • Control flow is the order in which a computer executes code in a script. In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page.
  • Control flow can be interrupted, however. We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all.
  • A conditional statement checks a specific condition(s) and performs a task based on the condition(s).
  • There are a few ways to create conditional statements in JavaScript.
    • Use if to specify a block of code to be executed, if a specified condition is true.
    • Use else to specify a block of code to be executed, if the same condition is false.
    • Use else if to specify a new condition to test, if the first condition is false.
    • Use switch to specify many alternative blocks of code to be executed.
      • When the computer encounters an if statement in a JavaScript file, it will check to see if the condition that we provide is truthy. If it is, then it will do whatever action we ask it to do in between the curly brackets. If the condition isn’t true or has a value that isn’t truthy, then the computer will ignore whatever is in between the curly brackets and move on.
    • An array allows us to evaluate hundreds of values at a time. While the if statement doesnt.
      • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.
      • A single array can store different types of data, such as strings, numbers, and so on.

    • Ternary Expression

    • The ternary operator is a conditional operator which evaluates either of two expressions – a true expression and a false expression – based on a conditional expression that you provide.
    • Syntax:
      • condition ? trueExpression : falseExpression
    • The ternary operator can be a good replacement for if statements in some cases. It allows you to write concise, cleaner, and easier-to-read lines of code if used well.
    • Learn properly:
      • https://www.freecodecamp.org/news/the-ternary-operator-in-javascript/

    The For Loop

    • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
    • In order to use the for loop, we need the shape's index and the array's length.
    • A for loop contains the following three important statements:
      • Keyword 'for'.
      • A three-part conditional statement.
      • A code block.
